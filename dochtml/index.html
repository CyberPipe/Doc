Атрибути до тега<button>можна застосувати будь-який уныверсальний атрибут, а також спуцифічні

autofocus- робить так, щоб при завантаженні сторінки кнопка вже була вибрана та підсвічена, так що її можна натиснути з клавіатури Enter.

disabledробить кнопку неактивною: натиснути на неї не можна, по дефолту відображається сірим кольором.

form— зв'язує кнопку будь-якої форми в документі через ID. Якщо ID не вказано, то з попередньою формою.

formaction— задає URL-адресу, на яку надсилаються дані після натискання на кнопку

formenctype — визначає формат файлу, який користувач може надіслати, натиснувши кнопку. У нього є три значення:

application/x-www-form-urlencoded: стандартне значення, замість пробілів ставиться +, а символи, на зразок російських букв, кодуються їх шістнадцятковими значеннями (наприклад, %D0%9F%D0%B5%D1%82%D1%8F замість Петя 🤡);

multipart/form-data- використовується, щоб надсилати файли, дані при цьому не кодуються;

text/plain- літери та інші символи не кодуються, замість пробілів з'являється +.


formmethod Повідомляє браузеру, яким HTTP-методом потрібно передавати дані на сервер. Є два методи:

post- Упаковує дані форми в тіло повідомлення і передає їх на сервер, обсяг даних залежить від налаштувань сервера;

get— додає дані з форми прямо в URL-адресу після знака «?», наприклад, тут пари «ім'я=значення» поділяються символом «&», а обсяг даних не може перевищувати 4 Кб. Якщо додати цей атрибут до кнопки, він перевизначить атрибут у всієї форми.https://site.ru/doc/?name=Ivan&age=27method


formnovalidate- Булевий атрибут, працює як novalidateдля тега

.

formtarget- Показує дані, які ввів користувач. Де браузер відкриє результат, залежить від ключового слова:

_self: показує дані у поточному вікні. Це значення використовується за умовчанням.

_blank: показує дані у новому вікні браузера - його використовують найчастіше.

_parent: показує дані всередині батьківського елемента кадру, а якщо такого немає, завантажує в поточному вікні.

_top: скасовує всі батьківські кадри та завантажує сторінку у повному вікні браузера. Якщо батьківських кадрів немає, то показує дані у поточному вікні.


name- Унікальне ім'я кнопки. Передається разом із даними, які надсилає користувач.

type- Задає дію кнопки при натисканні:

submit: надсилає дані на сервер. Це також значення за промовчанням.

reset: видаляє введені дані з форми.

button: просто кнопка Дію для неї можна задати через скрипти.


value— задає вихідне значення кнопки, яке надсилається на сервер разом із даними користувача.

Тег <input> дозволяє створювати інтерактивні елементи на сайті - поле для введення тексту, кнопка, повзунок, перемикач та інші

<!-- Типи для введення різних даних форматів-->

Тип елемента <input>

text - поле для ввода текста (в один рядок)

email - поле для ввода емейла. Браузер провірить чи є в ньому знак @ і домен

search - поле для пошуку.

tel - поле для ввода телефона.

url поле для ввода URL-адруса.

password - ввод пароля. Символи будуть замінятися на ***. Можна задати мінімальну і максимальну кількість символів.

number- ввод числа

<!-- Типи для кнопок -->

submit - просто кнопка для відправки форми

button - звичайна кнопка

image- кнопка «Відправки» в виді картинки. Щоб задати адрес картинки використовуйте атрибут src, а для альтернативного текста - атрибут alt. Щоб скоректувати розмір використовуйте height і width.

reset - кнопка яка скидує все що користувач ввів в поля форми раніше.

<!-- Типы для ввода дат-->

date - ввод дати без часуж: рік, місяць і дата.

detetime-local - ввод дати і часу в поточному часовому поясі, коли зони UTC не вказуються

month - ввод місяця і року, без указання часового пояса.

week - ввод номера тижня

time - ввод часу без вказування часового пояса.

<!-- Едементи форм -->

chackbox - галочки або чекбокс, з декільками варіантами на вибір

radio - кругла кнопка - переключатель дя вибору одного з декількох варіантів.

range - повзунок для вибора чисел із заданного діапазону.

color - віджет для вибора коляру. Інколи його називають колопикур (colorpicker)

file - Вибір файла для відправки
